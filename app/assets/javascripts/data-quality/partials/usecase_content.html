Annual Benchmarking Data Quality Use Case

Cities across the United States are instituting energy benchmarking initiatives to understand how much energy their commercial buildings are using. By targeting the highest energy users with strategies to improve energy efficiency, they can progress towards their emissions reduction goals and reduce costs for building owners.

There are already over 20 cities, 20 districts, and 200 corporations managing benchmarking initiatives and the list is growing each month. In fact, every city in the State of California is required to begin benchmarking commercial buildings in 2018 and over 7,000 cities around the world are adopting similar commitments through the Global Covenant of Mayors.

US Cities with Benchmarking Initiatives: http://www.imt.org/resources/detail/map-u.s.-building-benchmarking-policies

How are cities currently managing energy benchmarking program data?

In order to effectively develop energy efficiency strategies, cities begin by collecting data for all buildings of interest. The most common approach for managing the process of data collection is with the publically available EPA Portfolio Manager . Building owners create an account on EPA Portfolio Manager and populate the data fields required to comply with the city benchmarking program. Once building owners finish gathering the data, they share it with the city, which exports the data using a Portfolio Manager Report Template for analysis.

Graphic: Building (Data Collection) → Portfolio Manager (Data Sharing)  → CSV (Data Analysis)

Since the first public benchmarking program in 2009, cities have established effective guidelines for standardizing the process of data collection using EPA Portfolio Manager. However, after collecting the data, each city has a different way of analyzing it for setting effective energy efficiency strategies.

Through our engagement with several cities (Chicago, DC, Tacoma, Eugene, Providence, and more…), Maalka has discovered a common need for ensuring benchmarking program data is accurate, complete, and interoperable. To address this need, we developed the Data Quality Tool to allow municipalities to easily and cost effectively:

  1. Import EPA Portfolio Manager data
  2. Map the data to a BEDES compatible format
  3. Automatically identify data anomalies across their benchmarking program
  4. Flag possible errors in data related to building characteristics, operations, and energy-use.

Blue Question: What kind of data validation does the tool conduct?

The Maalka Data Quality Tool performs data validation on building characteristics and energy-use parameters based on the most recent data obtained by the EPA for buildings across the United States (https://www.energystar.gov/buildings/about-us/research-and-reports/portfolio-manager-datatrends). To provide the most useful insights, the data validation is based on building-type specific data. For example, the following plot and table can be observed in the Energy Star Portfolio Manager Data Trends document for an Office Building:

Source: https://www.energystar.gov/buildings/tools-and-resources/datatrends-energy-use-office-buildings


The Data Quality Tool automatically identifies all the buildings in the uploaded Portfolio Manager template and assesses values for associated building and energy-use characteristics. For example, an office building will have data validation for fields related to:

1)   Gross Floor Area

2)   Number of workers

3)   Number of Computers

4)   Weekly Operating hours

5)   Site Energy Use Intensity (EUI)

6)   Source EUI

Values for each building are compared against ranges that span the 5th and 95th percentiles of each parameter’s distribution for similar building types in the CBECS dataset, as observed in the above table above for office buildings. Buildings with data that falls outside of these ranges will be automatically flagged so that cities can review the data to ensure that it is accurate. This process prevents data entry errors and faulty information from being included in data analysis so cities can be confident in their energy efficiency improvement strategies.

Blue Question: How should I interpret the results of the Data Quality Tool?


Validation for each field is thus automated and presented to a user in this intuitive manner, so that users can instantly assess data quality and completeness for large portfolios of buildings. It is important to note, however, that the Maalka Data Quality Tool should be used as part of a broader process around data validation. A few important considerations for users:


1)   If a data field does not pass validation it does not mean that it is wrong. The following are possible reasons for failed validation:


  1. Data is missing
  2. Data is in an incorrect format (e.g. Postal Code is not Numeric)
  3. Data is out of normal range for a specific building type


2)   Data could be accurate and out of normal range, like in the above example for Number of Weekly Operating Hours, meaning that the building owner reported a very high or low number. Rather than discarding the data point, program managers should ask the building owner who supplied the data to confirm the value.


3)   Data could be missing intentionally. The Data Quality tool checks whether a building reported both Electricity and Natural Gas data. If it is missing either, then the tool flags it as a potential missing value. However, a building might not a Natural Gas account because it heats through an electrical system. Program managers should ask the building owner who supplied the data to confirm that the missing value was intentional.


4)   Data that does not pass validation may not be important to the program. For instance, in the above example the data field for Number of Weekly Operating Hours does not pass validation. If a program is only interested in checking Source EUI data, then a user can effectively ignore the fact that other values do not pass.


5)   Missing data may mean that data was not reported correctly. Program managers should ask the building owner who shared the data to resubmit with more complete information.



Tip: Keep a note next to your workspace of any buildings that have special facilities so as you work through the validation process you can identify them quickly and move on.


Blue Question: Can I export the results of the Data Quality Tool?


Yes! In order to process the results of the Maalka Data Quality Tool independently, users can download an EXCEL compatible CSV file by clicking the ‘Download CSV’ button.

 

The Data Quality Tool CSV export serves two main purposes:


1)   Allows users to analyze the data quality results in Microsoft Excel so that buildings (rows) and fields (columns) can be removed, filters, and visualized. This allows data validation to be easily integrated into periodic workflows so that program managers can have a good feeling for the quality of data across a portfolio as it becomes available.


2)   Translates all the Portfolio Manager field names into a BEDES compatible format for interoperability. The Building Energy Data Exchange Specification (BEDES) format is maintained by the US Department of Energy and provides a standard dictionary of terms for managing building related data. By translating the data from the template into a BEDES compatible format, users can shared data with external systems that align with the BEDES standard for rapid analysis and visualization across programs!
